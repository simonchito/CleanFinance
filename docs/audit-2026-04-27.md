# Auditoría 2026-04-27

## Alcance

Revisión integral del repo Flutter CleanFinance: arquitectura, tests, documentación, ARB/i18n, seguridad, Android, dependencias, robustez funcional y calidad de código.

## Hallazgos por prioridad

### Crítico

- No se detectaron fallos críticos reproducibles: `flutter analyze` y `flutter test` pasaban antes de aplicar cambios.

### Alto

- Portugués figuraba como idioma soportado (`pt`) pero el ARB estaba incompleto y mezclaba textos en español. Se retiró del set soportado para evitar una experiencia mitad traducida. `pt` y `pt_BR` ahora se normalizan a `system`.
- Logs técnicos de startup, SQLite, secure storage, auth y biometría podían imprimir rutas, errores o stack traces fuera de debug. Se restringieron a `kDebugMode`.

### Medio

- `.gitignore` ignoraba `android/`, lo que podía ocultar cambios relevantes de configuración Android. Se quitó ese ignore amplio y se agregaron ignores explícitos para secretos de firma.
- Documentación desactualizada: versión de DB, `compileSdk`, idiomas soportados y signing release no reflejaban completamente el código actual.

### Bajo

- Faltaba un test que protegiera la paridad de ARB y el set real de idiomas soportados.
- Dependencias con versiones nuevas disponibles: se documenta como evaluación separada porque la suite está verde y varias actualizaciones son saltos mayores.

## Archivos tocados y motivo

- `lib/core/localization/app_locale_mode.dart`, `lib/core/constants/app_constants.dart`, `lib/app/app_strings.dart`, `lib/features/finance/presentation/screens/settings_screen.dart`: limitar idiomas a `es`/`en`.
- `lib/l10n/app_es.arb`, `lib/l10n/app_en.arb`, `lib/core/localization/generated/*`: regenerar l10n sin portugués.
- `lib/main.dart`, `lib/core/database/app_database.dart`, `lib/core/security/*`, `lib/features/auth/presentation/auth_controller.dart`: restringir logs técnicos a debug.
- `.gitignore`: permitir versionar configuración Android y excluir secretos concretos.
- `test/app_locale_mode_test.dart`, `test/l10n_consistency_test.dart`: cubrir normalización de idiomas y paridad ARB.
- `README.md`, `docs/build-and-run.md`, `docs/database.md`, `docs/security.md`, `docs/dependencies.md`, `docs/google-play-checklist.md`: actualizar documentación real.

## Riesgos pendientes

- SQLite local sigue sin cifrado a nivel archivo.
- Privacidad de montos no se aplica de forma global en todas las pantallas.
- Soporte real de Web/Desktop e iOS requiere validación de runtime.
- Dependencias mayores nuevas deben evaluarse con una tarea dedicada.
